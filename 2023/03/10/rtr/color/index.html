色度学 ( Colorimetry )

大多数的光包含许多不同的电磁波波长，可以通过光谱能量分布 spectral power distribution (SPD) 进行可视化。

上图显示了三种不同的 SPD 分布，值得注意的是虽然后面两个 SPD 的分布完全不同，但是我们人眼感知到的却是相同的颜色， 由此可见 SPD 和感知到的颜色并不是简单的一一对应的关系，色度学正是用于研究这两者之间的关系。

人眼的视网膜上有三种不同类型的接收器，各自对于不同的波长有不同的反应，因此只需要三个数字就可以精确描述人眼感知到的颜色信息， CIE 提出了一组测量颜色的标准条件并执行了相关的颜色匹配实验。

颜色匹配实验

实验一边使用三种几乎单调的颜色，能量分布分别集中在如下波长范围内：r - 645 nm，g - 526 nm，b - 444 nm，这三束光投射在白色屏幕上叠加在一起形成对应的颜色，另一边是用于测试对照的单一波长的颜色。观察者通过调节 r，g，b，这三组光的权重 [-1, 1] 直到两组颜色匹配为止 ( 负数的权重表示相对应的光投射在测试颜色上 )，一组测试结果如下：

显示编码

我们在计算光照、纹理或其他操作时应该在线性空间里进行，然而显示缓冲区和纹理使用的是非线性的编码，因此当我们在 shader 中计算完毕准备输出时需要进行伽马矫正。

display transfer function 描述了显示缓冲区中的数值和输出辐射率之间的关系，也称为 electrical optical transfer function (EOTF)，属于硬件自身的特性，对于电脑显示器、电视机、电影放映机等都有不同的标准。

当编码线性颜色用于显示时，我们的目标是抵消 display transfer function 的效果，例如，如果最后计算出的颜色数值翻倍了，我们希望输出的辐射率也翻倍。为了实现这一目标，我们在最后输出时应用 display transfer function 的逆变换，该过程即为伽马矫正。

电脑显示器的标准 display transfer function 定义为 sRGB，在读取纹理或者写入颜色缓存时 GPU 一般会自动应用适合的 sRGB 转换，如果需要手动应用 sRGB，也可以使用如下等式：

HDR 编码

standard dynamic range (SDR) 显示器一般使用 sRGB 显示标准，而 HDR 使用的是 Rec. 2020 和 Rec.2100 标准，其中 Rec. 2020 定义了一个具有更宽色域范围的颜色空间，Rec. 2100 定义了两种非线性显示编码：perceptual quantizer (PQ) 和 hybrid log-gamma (HLG)

在应用侧，有三种方式可以将图片转换成 HDR 显示

  1. HDR10 : 在 PC 等 HDR 显示设备上广泛支持，格式为每个像素 32 位 (每一个 RGB 通道 10 位无符号整数，2 位 alpha)，使用 PQ 非线性编码和 Rec. 2020 颜色空间。每个 HDR10 显示模型使用各自独有的 tone mapping，没有相应标准和文档
  2. scRGB (linear variant) : 只支持 Windows 操作系统，格式为每个通道 16 位，储存线性的 RGB 数值，可以适配 HDR10 显示设备并且向后兼容 sRGB
  3. Dolby Vision : 专有的格式没有广泛的支持，格式为每个通道 12 位，使用 PQ 非线性编码和 Rec. 2020 颜色空间，具有标准化的内部 tone mapping，但是没有文档

Tone Mapping

Tone mapping 或者说 tone reproduction 是将 scene radiance values 转换为 display radiance values 的过程，该变换称为 end-to-end transfer function 或者 “scene-to-scene transform”，image state 这一概念是理解 tone mapping 的关键。

有两种基本的 image state

  1. Scene-referred images : 根据 scene radiance values 定义
  2. Display-referred images : 根据 display radiance values 定义

image state 和具体编码是无关的，在每种状态中图像都可以编码为线性或非线性的，下图显示了 image state，tone mapping 和 display encoding 在整个管线中的位置

Tone mapping 可以认为是一种 image reproduction，image reproduction 的目标是：

“create a display-referred image that reproduces – as closely as possible, given the display properties and viewing conditions – the perceptual impression that the viewer would have if they were observing the original scene.”

此外还有另一种类型 image reproduction，preferred image reproduction 的目标是：

“create a display-referred image that looks better, in some sense, than the original scene.”

Exposure ：在 tone reproduction 变换之前对 scene-referred 图像执行线性的缩放操作

全局 tone mapping : 对所有像素执行同样的变换

局部 tone mapping : 基于周围的像素和一些其他因素，对每个像素执行不同的变换

Tone reproduction 变换

一般表示为一维曲线函数，将 scene-referred 输入值映射为 display-referred 输出值，可以应用于每个独立的 R，G，B 或者 luminance 上。

当应用于每个独立的 R，G，B 时，输出结果将自动位于显示的色域范围内，然而对 RGB 通道 执行非线性操作将导致色调 (hue) 和饱和度 (saturation) 的偏移，现代 tone 变换会应用额外的 RGB 调整来减少色调的偏移。

当应用于 luminance 上时，色调和饱和度偏移可以减少或避免，然而输出的 display-referred 颜色可能会在显示的色域范围之外，这时需要重新映射回范围内。

关于 tone mapping 的一个潜在的问题 ：对 scene-referred 像素颜色应用一个非线性的函数会导致反走样技术相关的问题。

实时渲染中使用的 Tone transformation ：

  1. The Reinhard toon reproduction operator : 最早使用的方法之一，其保持大多数暗部数值不变，而亮部数值渐进到白色
  2. Duiker 提出了一个类似于 Kodak film response curve，随后 Hable 对其进行修改增加更多用户控制，并使用在神秘海域2中
  3. Day 提出了一个 sigmoid tone curve，并使用在使命召唤中
  4. ACES 标准 ：将 screen-to-screen 变换分为两部分，第一部分是 reference rendering transform (RRT)，将 screen-referred 数值变换到一个标准的、设备无关的输出空间 (称为 output device transform color encoding specification OCES) 中的 display-referred 数值；第二部分是 output device transform (ODT)，将 OCES 中的颜色数值变换到一个特定的显示设备和观看条件中。Unreal 引擎默认启用 ACES，同时Unity 引擎也支持

Exposure

计算 exposure 的一类常用技术依赖于对 scene-referred luminance 的分析，通常对前一帧进行采样分析。准则之一是使用 log-average scene luminance，其缺点在于对异常数值太敏感 (少量明亮的像素会影响整个帧的 exposure)；为了缓减这一问题，又提出了使用 histogram of luminance values

然后上述这些基于像素 luminance 来决定 exposure 的方法是错误的，正确的做法是基于光照本身 (不考虑表面反射的效果) 来决定 exposure，这是因为摄影的曝光是用来抵消光照的，从而能够显示物体的表面颜色，也确保正确的数值传入 tone 变换。合金装备 5 中使用了基于光照强度的 exposure 系统，在许多游戏中，基于已知的场景光照数值，对环境中的不同部分手动设置静态的 exposure 强度

Color Grading

前面提到的 preferred image reproduction 一般通过 color grading 对图像颜色进行创造性的处理来实现，其实现过程如下：先对一个示例场景图像交互式地处理变换其颜色，直到获得想要的创造性的视觉效果，然后对其余所有图像进行同样的操作序列。

Selan 提出了 color lookup table (LUT) ，使用输入 R，G，B 作为 x-，y-，z- 坐标就可以在 LUT 中查询到其变换后的颜色，有两种执行 color grading 的方法:

  1. 在 display-referred 图像数据上执行
  2. 在 scene-referred 图像数据上执行

scene-referred 方法具有更高的视觉质量，另外通过将 tone mapping 曲线也并入 LUT 中也可以减少一些计算；在 LUT 查找之前，scene-referred 数据必须被重新映射到 [0, 1] 范围。
